Actual-Sim True-Oracle Remote Handoff

A renderer-capable Linux/GPU handoff plan that reuses existing environments. It runs a cheap zero-action probe first, then runs SmolVLA only if pose/camera capture is proven.

Single command

ROOT_OUTPUT_DIR=_workspace/checkpoints/checkpoint_24_actual_sim_true_oracle_two_stage \
EPISODES=1 \
STEPS=12 \
MIN_STRICT_STEPS=10 \
sh scripts/run_actual_sim_true_oracle_probe_then_policy_cp24.sh
1

Reuse environment

No pod creation, no pod stop. Run only inside an already-approved renderer-capable environment.

2

Probe first

Stage 1 uses affordance_oracle_probe and zero actions, avoiding SmolVLA model loading.

3

Actual RGB

Probe must capture actual ManiSkill RGB frames before any policy claim is considered.

4

Object pose

Probe must produce object_pose_xyz from the real environment.

5

Camera metadata

Probe must produce camera intrinsic/extrinsic metadata keys from the same timestep.

6

Projection mode

Strict samples require projected_object_pose, not center fallback.

7

Gate policy stage

SmolVLA policy stage runs only after probe has at least 10 strict samples.

8

Policy manifest

Policy stage should produce smolvla_affordance_true_oracle_steps.json.

9

Summary file

Two-stage result lands in two_stage_summary.json for quick handoff review.

10

Probe fail

If probe fails, do not debug SmolVLA yet. Renderer/env metadata is still the blocker.

11

Policy fail

If probe passes but policy fails, then the remaining blocker is model loading or policy input integration.

12

Claim boundary

Even passed true-oracle policy input is an upper-bound diagnostic, not final benchmark improvement.
