Remote True-Oracle Evidence Pack

A single non-destructive command for renderer-capable environments: preflight, two-stage true-oracle gate, result import, dashboard rebuild.

Copy/paste command

ORACLE_OVERLAY_ROOT=_workspace/runpod_results/affordance_overlay_validation_20260606T1955Z \
ROOT_OUTPUT_DIR=_workspace/checkpoints/checkpoint_24_actual_sim_true_oracle_two_stage \
EPISODES=1 \
STEPS=12 \
MIN_STRICT_STEPS=10 \
PYTHON_BIN=python3 \
sh scripts/run_remote_true_oracle_evidence_pack.sh
1

No pod lifecycle

Does not create, stop, delete, resize, or mutate pod lifecycle state.

2

Preflight first

Checks Python, imports, GPU/Vulkan visibility before attempting simulation.

3

Probe before policy

Runs zero-action affordance_oracle_probe before any SmolVLA policy stage.

4

Policy gated

SmolVLA true-oracle path only runs if probe reaches the strict sample threshold.

5

10+ strict samples

Default gate is MIN_STRICT_STEPS=10 with STEPS=12.

6

Result importer

Renders two_stage_summary.json into HTML and manifest automatically.

7

Dashboard rebuild

Runs the repo milestone rebuild so the latest status appears in one place.

8

Artifact audit

Regenerates completeness audit after evidence collection.

9

Pack summary

Writes remote_true_oracle_evidence_pack_summary.json.

10

Blocked is useful

If it blocks, stage-specific status tells us whether the problem is preflight, probe, or policy.

11

Claim boundary

The runner does not prove Tier O; only generated strict manifests can do that.

12
