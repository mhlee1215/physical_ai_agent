Actual-Sim True-Oracle Capture Contract

The implementation now preserves a cleaner path from actual simulation observation to oracle overlay metadata. This report defines the 10+ sample gate that must pass before Tier O is claimable.

patchedPolicy-input overlay path now uses augmented oracle observation when env state is available.
not runManiSkill/SmolVLA execution was not launched from this Mac turn to avoid dependency/download side effects.
gate openNeed strict_true_oracle_step_count >= 10 from actual simulation.
1

Actual RGB

Every strict sample needs a raw frame from the same action-input observation.

2

Overlay RGB

The overlay frame must be generated from that same observation, not a later step.

3

Object pose

object_pose_xyz must be present in oracle_affordance.

4

Camera metadata

camera_metadata_keys must contain usable intrinsic/extrinsic fields.

5

Projected mode

mode must equal projected_object_pose, not center fallback.

6

Episode id

Each record must preserve the simulator episode id for seed/task matching.

7

Step id

Each record must preserve the action step that produced the policy input.

8

Success flag

Benchmark success remains the simulator/environment success flag.

9

Privileged status

True oracle is an upper bound, not a deployable no-privilege method.

10

Import path

Passing manifest should be imported with scripts/import_actual_sim_true_oracle_milestone.sh.

11

Failure mode

If projection falls back to image center, camera or object pose extraction is still incomplete.

12

Minimum count

Fewer than 10 strict samples is insufficient for the milestone dashboard.
